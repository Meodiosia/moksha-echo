# Moksha Echo · 解脱回声

> 复仇与轮回主题的 2D 横版动作 BOSS 战 · HTML5 单文件原型

---

## 一、项目简介

**Moksha Echo（梵语「解脱」+ 英语「回声」）** 是一款基于 HTML5 Canvas 2D 的等距视角动作游戏原型，灵感来自《哈迪斯》《空洞骑士》《忍者龙剑传》系列。

**核心体验**：
- 单角色（**飞剑使**） vs 单 BOSS（血主 **Lucia**）的 1v1 极限对决
- **11 个技能 + 长按蓄力 + 飞剑环绕**的复合战斗系统
- BOSS 三阶段渐进狂暴 AI，**15+ 招式池**含远程追踪弹、分身投剑、火柱、锁链
- 支持 PC 键盘 + 移动端虚拟摇杆 + 自动 AI 战斗
- 手绘像素 + 等距 3/4 俯视 + 哈迪斯地狱画风

**画面主题**：十字血祭祭坛 · 红黑色调 · 大量血雾粒子与剑气光效

---

## 二、画面与玩法

### 主角：飞剑使

- 武器：8 把环绕飞剑（自动每秒 +1，上限 8）
- 战斗手感：消耗飞剑射出 → 飞剑命中后自动飞回
- 11 个技能涵盖近战/远程/AOE/位移/防御反弹/大招

### BOSS：Lucia 血主

- **3 阶段**血量进程：100% → 70% → 28%（直接 P3 起手）
- P3 形态：高速、追击、招式池含 15+ 种
- 招式举例：
  - **血祭轮回**（4 分身轮番冲锋）
  - **血噬**（3 颗追踪弹缓速锁定）
  - **血影双煞**（玩家两侧召分身投 6 把巨剑十字夹击）
  - **血祭** / **血色风暴** / **狂袭** / **裂魂** / **血柱** ...
- **主动格挡**：P3 期间 0.5s 一次 55% 概率主动挡 + 弹反
- **中心引力**：双方都被无形场拉回十字祭坛中心，**避免边缘缠斗**

### 关卡

- **十字祭坛**（72×48 tile）：地图三主题，地下血祭氛围
- 玩家西侧出生，BOSS 东侧出生，自动汇聚中心 PK

## 三、操作方式

### PC 键盘

| 键 | 功能 |
|---|---|
| **WASD / 方向键** | 移动（双击触发奔跑）|
| **J** | 普攻（飞剑射击）|
| **K / Space** | 冲刺 |
| **I** | 防御 |
| **L** | 长按蓄力剑阵 |
| **O** | 长按蓄力光波 |
| **P** | 天落剑雨 |
| **N** | 召剑爆发 |
| **M** | 飞剑风暴 |
| **U** | 大招 |
| **B** | 千机突刺（瞬步穿身）|
| **V** | 剑阵齐射 |
| **Tab** | 切换 AI 自动战斗 |

### 移动端

- **左下虚拟摇杆**：点哪生哪的动态摇杆
- **右下技能盘**：3 圈分布
  - 第 1 圈：普攻 J（最大）
  - 第 2 圈：5 个常用技能（K/I/O/N/M）半径 95
  - 第 3 圈：5 个高阶技能（P/L/U/V/B）半径 150
- **左上 AI 按钮**：一键开启自动战斗
- **右上 BOSS 按钮**：暂停 BOSS AI（练习用）
- 圆形 CD 进度遮罩 + 按下浮雕反馈

## 四、技术架构

### 总体

- **前端纯静态**：HTML + JS + PNG 帧序列
- **无构建依赖**：浏览器直接打开（需 HTTP 服务器，Python `http.server` 即可）
- **公网测试**：`cloudflared tunnel`（零注册临时 URL）

### 渲染

- HTML5 Canvas 2D，固定 800×560 内部分辨率
- **等距视角**（ISO_Y_SCALE = 0.62，哈迪斯式 3/4 俯视）
- 像素风（image-rendering: pixelated）
- 帧动画系统（每动画 N 帧 PNG 序列）
- **粒子 + 冲击波 + FX 形状池**（crescent / abyss / xslash / speedline 等可复用）
- 振屏 / 时停 / 慢动作 / 残影系统

### 物理与战斗

- 马里奥式加速度 + 强摩擦
- 分轴碰撞（moveAxis）
- Y 轴 1px 容差对齐（避免「同 X 不同 Y 打空气」）
- 中心引力场（双方拉回中心）
- HP × 100 数值（10000 / 150000，每秒回 800 HP）

### AI 系统

- **玩家 AI**（player_ai.js）：行为树 + 虚拟输入注入（_AI_IX/_AI_IY），9 级技能优先级
- **BOSS AI**（lucia_boss.js）：阶段权重池 + 距离判定 + 最近使用排序，3 阶段差异化决策

### 移动端适配

- IS_MOBILE 自动检测（UA + 触屏 + 屏幕尺寸）
- cover 模式 canvas 缩放（铺满屏幕，超出裁切）
- 触摸事件 → keyboard 事件桥接（fireKey）
- 性能降级：粒子上限减半、残影关闭、ambient 渐变改纯色

### 设计原则

1. **无构建工具**：浏览器直接跑，零依赖

3. **视觉冲击**：粒子 + 冲击波 + FX 形状叠加
4. **不重复造轮子**：BOSS 复用 player 的 FX 池

## 七、运行方法

### 本地运行

```bash
cd game
py -3 -m http.server 8765
# 浏览器打开 http://localhost:8765/demo-3c.html
```

### 公网测试

```bash
cloudflared tunnel --url http://localhost:8765
# 复制输出的 https://*.trycloudflare.com URL
# 任意设备打开（手机/平板）
```

### 编辑器工具

- `map_editor3.html`：地图编辑器（绘制 walkable / 出生点）
- `skill_editor.html`：技能参数调节器（导出 JSON）
- `mask_editor.html` / `mask_editor2.html`：地面碰撞掩码编辑

## 九、技术亮点

1. **飞剑系统**：8 把剑独立状态机（orbit/shoot/return/hover），命中 → 自动归位 → 环绕，单一对象池服务全部技能
2. **BOSS 招式权重池**：阶段差异化 + 最近使用衰减 + 距离判定，避免重复连发同招
3. **共享 FX 池**：玩家与 BOSS 共用 `fxShapesPool`，动画一致性 + 性能上限保护
4. **虚拟输入桥接**：AI / 移动端 / 键盘三种输入源统一注入主循环 ix/iy 接口
5. **Cover 模式 + GPU 缩放**：手机端绕过 CSS reflow，规避 Safari/微信内核闪屏问题
6. **CD 圆锥渐变**：纯 CSS conic-gradient，无需 canvas 二次渲染
